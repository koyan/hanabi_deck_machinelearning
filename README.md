# 🎴 Hanabi Card Recognition System (v1.0)

A Deep Learning pipeline designed to recognize a custom 35-card Hanabi set (including Standard Colors, Rainbow, and Black suits). This project uses **MobileNetV2** fine-tuning and is optimized for small, imbalanced datasets.

## 🚀 Purpose
To provide a fast, repeatable environment for training an AI model that can identify Hanabi cards from photos. It includes a built-in "Data Pipeline" that automatically resizes 4K images into AI-ready formats to keep training fast.

---

## 🏗️ Project Structure
```text
.
├── hanabi_dataset/          # 📥 SOURCE: Drop your high-res (4K) photos here
├── hanabi_dataset_small/    # ⚡ AUTO: Resized images (generated by script)
├── prepare_data.py          # Script: Syncs & Resizes images
├── train_hanabi.py          # Script: The "Brain" - Trains the model
├── error_report.py          # Script: Audit - Tells you which cards to fix
├── verify_model.py          # Script: Visual check - Saves a 3x3 results grid
├── Makefile                 # Shortcuts for Docker commands
└── Dockerfile               # The environment (TensorFlow, Pillow, etc.)
```

## 🛠️ Usage Instructions

### 1. Initial Setup
Ensure you have **Docker** installed. Build the engine once:
```bash
make build
```

### 2. Enter the Lab
To do any work, enter the Docker bash session. This mounts your local folder into the container so any code changes you make on your computer are reflected instantly.
```bash
make bash
```

### 3. The Workflow (Inside the Container)

#### Step A: Sync your Data
Every time your team adds new photos to \`hanabi_dataset/\`, run this. It will only resize **new** images, saving time.
```bash
python prepare_data.py
```

#### Step B: Train the Model
This will run for 30 epochs and output a \`hanabi_model_v1.tflite\` file and an \`accuracy_plot.png\`.
```bash
python train_hanabi.py
```

#### Step C: Run the Audit
After training, check which cards the model is struggling with. 
```bash
python error_report.py
```

---

## 📊 Understanding the Reports

### The "Action Plan"
The \`error_report.py\` script will show you a table of failures.
* **Failure % > 50%:** These cards are "invisible" to the AI. Add 20+ photos of these immediately.
* **Most Confused With:** If \`Red 1\` is confused with \`Red 2\`, take closer, sharper photos of the numbers.

### Visual Verification
Run \`python verify_model.py\` to generate \`validation_results.png\`. This shows 9 random cards the model has **never seen before** and its guesses. Green titles = Correct, Red titles = Mistake.

---

## 💡 Best Practices for Photos
1. **Quantity:** Aim for at least **30-50 photos** per card for high accuracy.
2. **Variety:** Take photos from different angles, distances, and lighting (shadows vs. bright light).
3. **Backgrounds:** Take photos on the table you actually play on!
4. **The "Black" Suit:** Since these have lower contrast, ensure some photos are taken in very bright light.
